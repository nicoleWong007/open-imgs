#### Step 2: Domain Models
- Define data classes in `shared/commonMain/domain/model/`
- `Photo` (id, path, dateTaken, size, mimeType, width, height, orientation, isFavorite)
- `Album` (id, name, coverPhotoId, photoCount, createdAt)

- `StorageCategory` (type, itemCount, totalSize, displayName)
- `PremiumStatus` (isPremium, expiryDate, features)

**Tests:** Model serialization round-trips, data class equality

#### Step 3: SQLDelight Schema
- Create `.sq` files in `shared/commonMain/sqldelight/`
- `Photos.sq` — CRUD for photo metadata, embedding storage, hash storage
- `Albums.sq` — album CRUD, photo-album membership
- `Duplicates.sq` — duplicate groups, resolution state
- `SearchIndex.sq` — embedding vectors, search metadata
- `Premium.sq` — subscription state, feature usage tracking
- Configure SQLDelight Gradle plugin for Android + iOS native drivers

**Tests:** Schema migration tests, CRUD operations, vector distance queries

#### Step 9 (parallel): PremiumManager
- Add RevenueCat KMP 2.10.2 dependency
- Implement `PremiumManager` in `shared/commonMain/data/premium/`
- Feature gating: `checkFeatureAccess(feature) -> Boolean`
- Free tier limits: 5 albums, 10 searches/month, 5 duplicate deletes/day
- Handle: purchase, restore, expiry, offline grace period

**Tests:** Feature gating logic, subscription state transitions, expiry handling

### Phase 2: Platform Integration + Core Logic (Lane A + B)

#### Step 4: Platform Photo Access (expect/actual)
- `expect class PhotoLibraryAccess` in commonMain
- iOS actual: PhotoKit wrapper (PHAsset fetch, PHImageManager thumbnails)
- Android actual: MediaStore queries, ContentResolver thumbnails
- Handle permissions, change notifications, incremental sync
- **Critical**: Live Photo detection on iOS (PHLivePhoto pairing)

**Tests:** Platform integration tests (need emulators), permission flow

#### Step 5: PhotoRepository
- Implement in `shared/commonMain/domain/repository/`
- Paginated photo loading (page size: 100, sorted by date)
- Album CRUD with free tier limit enforcement
- Photo-album membership management
- Background sync coordinator (detect new/deleted photos)

**Tests:** Pagination, album limit enforcement, idempotent add-to-album

#### Step 6: DuplicateDetector
- Implement pHash + dHash in `shared/commonMain/domain/dedup/`
- pHash: DCT-based 64-bit hash (resize to 32x32, compute DCT, take top-left 8x8)
- dHash: Gradient-based 64-bit hash (resize to 9x8, compute horizontal gradients)
- Hamming distance comparison: distance ≤ 10 = potential duplicate
- Live Photo handling: detect paired image+video, always keep or delete both
- Group duplicates by similarity threshold
- Background indexing: compute hashes for new photos incrementally

**Tests:**
- Hash computation for various formats (JPEG, PNG, HEIC)
- Hamming distance correctness
- Live Photo pair integrity
- Near-duplicate detection (cropped, rotated)

#### Step 7: SearchEngine (CLIP)
- Add ONNX Runtime dependency (platform-specific native libs)
- Bundle CLIP ONNX model (all-MiniLM-L6-v2 INT8, ~22MB)
- Implement embedding generation in `shared/commonMain/data/search/`
- Store 384-dim vectors in SQLDelight (BLOB column)
- Vector similarity search: cosine distance in SQL or in-memory
- Query encoding: text → CLIP → 384-dim → find nearest
- Premium gating: free users get 10 searches/month

**Tests:**
- Embedding generation for sample images
- Query encoding correctness
- Vector search accuracy (known image → find it)
- Premium limit enforcement
- Corrupt model file → graceful degradation
- Empty string / very long input handling

#### Step 8: StorageAnalyzer
- Categorize photos: screenshots, duplicates, large videos, bursts, blurred
- Compute per-category size and count
- Batch delete with progress tracking
- Show "freed X GB" after deletion
- Android: internal + SD card detection

**Tests:** Category classification, batch delete atomicity, partial delete recovery

## Design Specification (from /plan-design-review)

### Navigation Architecture

**5-tab bottom navigation:** Photos | Albums | Search | Clean | Settings

- **Photos tab**: Primary. Photo grid by date, 4 columns default, pinch to zoom 3-5 cols.
- **Albums tab**: Album cards (2 columns), create/rename/delete, free tier limit indicator.
- **Search tab**: Smart search with CLIP, "getting better" progress during indexing.
- **Clean tab**: Merged duplicates + storage management. Hero stat "Free up X GB", category cards.
- **Settings tab**: Premium status, indexing progress, version info.

### Gesture Vocabulary

| Gesture | Action |
|---------|--------|
| Tap photo | Full-screen detail view |
| Long press | Select mode (multi-select) |
| Swipe left | Delete |
| Swipe right | Favorite |
| Pinch | Zoom grid density (3/4/5 cols) |
| Swipe up (detail) | Metadata sheet (EXIF, albums, size) |
| Swipe back | Return to gallery |

### Design System (create as DESIGN.md)

**Typography:** SF Pro Display/Text (iOS), Roboto Medium/Regular/Light (Android). Platform native.

**Colors:**
```
--color-background:     #FFFFFF / #000000
--color-surface:        #F2F2F7 / #1C1C1E
--color-text-primary:   #000000 / #FFFFFF
--color-text-secondary: #6E6E73 (WCAG AA 4.5:1 contrast)
--color-accent:         #007AFF (system blue)
--color-destructive:    #FF3B30
--color-success:        #34C759
--color-premium:        #FF9500
```

**Spacing:** 8pt grid. 8, 16, 24, 32pt.
**Cards:** 12pt corner radius, no shadow, flat surface. Only when card IS the interaction.
**Icons:** SF Symbols (iOS) / Material Symbols (Android). No custom icon set.
**Dark mode:** Supported at launch via system theme.

### Interaction States

```
FEATURE        | LOADING              | EMPTY                        | ERROR
───────────────|──────────────────────|──────────────────────────────|──────────────────────
Photos         | Skeleton grid        | "Photos appear here" +       | "Couldn't load photos.
               | (gray rectangles)    | "Open Camera" CTA            | Tap to retry."
Albums         | Skeleton cards       | "No albums yet" +            | "Couldn't load albums.
               |                      | "Create first album" CTA     | Tap to retry."
Search         | Pulsing dots         | "No photos match" +          | "Search unavailable.
               |                      | suggestions                  | Try again."
Clean          | "Scanning..."        | "Looking good! No junk 🎉"  | "Couldn't scan.
               |                      |                              | Check back later."
Photo Detail   | Blur preview         | N/A                          | "Photo couldn't load"
```

### Onboarding Flow

1. **Trust screen**: "Your photos. Your privacy." — bold text, clean background
2. **Privacy screen**: "Everything stays on your phone. No cloud. No tracking."
3. **Permission prompt**: "OpenImgs needs access to browse your photos"
4. → Gallery loads

### Delete Flow

- **Single photo swipe-delete**: No confirmation. Photo goes to 30-day system trash (recoverable).
- **Duplicate resolution**: Bottom sheet "Keep [photo] / Delete [N] photos"
- **Batch delete (Clean tab)**: "Delete X items? This frees X GB." Confirmation required.

### Search UX

- Debounced 300ms, results as you type
- First-run suggestion bubble: "Try: sunset, birthday, your pet's name"
- Partial index state: "Search is getting better! 12,340/50,000 indexed"

### Accessibility

- Touch targets: 44×44pt minimum (tabs 48×48pt, thumbnails 88×88pt)
- Dynamic Type / scalable fonts supported
- VoiceOver / TalkBack labels for all interactive elements
- Secondary text contrast: #6E6E73 on white (4.5:1 WCAG AA)
- Reduce Motion: disable photo fade-in, instant transitions
- RTL language support: flip layout direction

### Responsive

- **Phone (375-428pt)**: 4-col grid, 2-col albums, full-width clean cards
- **Tablet/iPad (768-1024pt)**: 6-col grid, 3-4 col albums, master-detail for albums
- **Foldable (Android)**: Expanded inner screen increases grid density

### NOT in Design Scope

| Face recognition UI | Deferred to v2 |

| Share sheet extensions | Out of scope |
